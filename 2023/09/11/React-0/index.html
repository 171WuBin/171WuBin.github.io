React
发布日期:   2023-09-11

React 可以改变你对可见设计和应用构建的思考。当你使用 React 构建用户界面时，你首先会把它分解成一个个 组件，然后，你需要把这些组件连接在一起，使数据流经它们。在本教程中，我们将引导你使用 React 构建一个可搜索的产品数据表。

1. 对 react 的理解 ？

React，用于构建用户界面的 JavaScript 库，只提供了 UI 层面的解决方案
遵循组件设计模式、声明式编程范式和函数式编程概念，以使前端应用程序更高效
使用虚拟 DOM 来有效地操作 DOM，遵循从高阶组件到低阶组件的单向数据流
帮助我们将界面成了各个独立的小块，每一个块就是组件，这些组件之间可以组合、嵌套，构成整体页面
react 类组件使用一个名为 render() 的方法或者函数组件return，接收输入的数据并返回需要展示的内容

2. 函数组件类组件的使用 ？组件的定义注意事项？

函数组件和类组件都可以用于构建 React 组件，函数组件通常用于简单的 UI 组件，类组件则更适合复杂的交互逻辑。在定义组件时，需要注意组件的 props 和 state 的定义和使用。

3. 什么是受控组件？什么是非受控组件？怎么改变？

受控组件，就是受我们控制的组件，组件的状态全程响应外部数据
非受控组件，就是不受我们控制的组件，一般情况是在初始化的时候接受外部数据，然后自己在内部存储其自身状态
非受控改受控：1.通过Refs获取dom节点，2.通过setState进行绑定和修改

4. 父传子、子传父、兄弟传参数？

父传子:
    1、父组件准备数据,父组件通过属性直接传递给子组件
    2、子组件通过props接收
子传父：
    1、父组件提供回调函数，传递给子组件
    2、子组件调用父组件中传递的回调函数
    3、将需要传递的参数，作为回调函数的参数进行传递
兄弟传值：
    状态提升：将共享状态写到最近的父组件里，由父组件管理状态
    父组件
        1、提供共享状态
        2、操作共享状态的方法
    兄弟组件的通讯
        1、通过props接受状态或者操作状态

5. refs?

React 中的 Refs提供了一种方式，允许我们访问 DOM节点或在 render方法中创建的 React元素
本质为ReactDOM.render()返回的组件实例，如果是渲染组件则返回的是组件实例，如果渲染dom则返回的是具体的dom节点

6. 新特性有哪些？并说出他们的用途及用法？

JSX 语法 它是一种JavaScript语法扩展，在React中可以方便地用来描述UI。
单向数据绑定 先把模板写好，然后把模板和数据（数据可能来自后台）整合到一起形成HTML代码，然后把这段HTML代码插入到文档流里面
虚拟 DOM  为了更好将虚拟的节点渲染到页面视图中，虚拟 DOM 对象的节点与真实 DOM 的属性一一照应
声明式编程 它表达逻辑而不显式地定义步骤。这意味着我们需要根据逻辑的计算来声明要显示的组件
Component

7. jsx 的本质是什么？jsx 语法注意事项？

JSX 本质上是一种语法糖，可以将 HTML 标签和 JavaScript 代码混合在一起，方便我们构建 UI 组件。需要注意的是，JSX 中的 class 属性需要写成 className，for 属性需要写成 htmlFor。

8. 类组件与函数组件里状态管理分别使用什么？

类组件可以通过 state 来管理组件状态，函数组件可以使用 Hooks 中的 useState 来管理组件状态。

9. 事件绑定的方式？

render方法中使用bind
render方法中使用箭头函数
constructor中bind
定义阶段使用箭头函数绑定

10. this 指向？ 普通函数，箭头函数

this 指向在普通函数中指向调用该函数的对象，而在箭头函数中指向该箭头函数定义时的对象。

11. 脚手架创建项目？

可以使用 create-react-app 等脚手架工具来创建 React 项目。

12. react 元素渲染？

React 元素渲染是指将 React 元素转换为 DOM 节点并插入到页面中的过程，可以通过 ReactDOM.render 方法来实现。

14. useCallback 和 useMemo 的区别？

useCallback 和 useMemo 都可以用于优化组件性能，前者用于缓存函数，后者用于缓存计算结果。两者的区别在于 useCallback 返回的是一个函数，而 useMemo 返回的是一个值。

15. 更新导致重新渲染的方式？

更新导致重新渲染的方式包括 props 和 state 的改变、父组件的重新渲染、forceUpdate 方法的调用等。

16. 浏览器渲染页面过程？

js放在head中会立即执行，阻塞后续的资源下载与执行。因为js有可能会修改dom，如果不阻塞后续的资源下载，dom的操作顺序不可控。
浏览器一边下载HTML网页，一边开始解析 解析过程中，发现<script>标签
暂停解析，网页渲染的控制权转交给JavaScript引擎
如果<script>标签引用了外部脚本，就下载该脚本，否则就直接执行
执行完毕，控制权交还渲染引擎，恢复往下解析HTML网页
html需要等head中所有的js和css加载完成后才会开始绘制，但是html不需要等待放在body后的js下载完成 就会开始绘制,因此将js放在body的最后面，可以避免资源阻塞，同时使静态的html页面迅速显示。

17. 导出导入？单个？ 多个？

导出和导入可以使用 ES6 的 export 和 import 语法，单个导出使用 export default，多个导出使用 export。

18. 类组件和函数组件的区别？

类组件和函数组件的区别在于类组件需要继承 React.Component，可以使用 state 和生命周期等功能，而函数组件则不需要继承，可以使用 Hooks 等功能。

19. useEffect useLayoutEffect 区别及参数？

useEffect可以让我们在函数组件中进行一些带有副作用的操作
useEffect 是异步执行的，而useLayoutEffect是同步执行的。
useEffect 浏览器完成渲染之后执行，而 useLayoutEffect 浏览器把内容渲染到界面之前执行

优先使用 useEffect，因为它是异步执行的，不会阻塞渲染
会影响到渲染的操作尽量放到 useLayoutEffect中去，避免出现闪烁问题
useLayoutEffect和componentDidMount是等价的，会同步调用，阻塞渲染
在服务端渲染的时候使用会有一个 warning，因为它可能导致首屏实际内容和服务端渲染出来的内容不一致。

20. context 使用 ?

Context 可以用于在组件树中共享数据，通过创建一个 Context 对象，然后在 Provider 组件中提供数据，子组件可以通过 Consumer 组件来获取数据。

21. 生命周期有几个阶段，分别是什么 ？执行顺序？ 调用时机？

React 生命周期有三个阶段：挂载阶段、更新阶段和卸载阶段。在挂载阶段，组件被渲染到页面上；在更新阶段，组件的 props 或 state 发生变化时会重新渲染；在卸载阶段，组件被从页面中移除。
constructor     初始化state状态或者在this上挂载方法
getDerivedStateFromProps    组件创建和更新阶段
render  用于渲染DOM结构
componentDidMount   render方法之后执行 此方法多用于执行一些数据获取，事件监听等操作
shouldComponentUpdate   到新的props或者state时都会调用
getSnapshotBeforeUpdate     render后执行
componentDidUpdate  组件更新时触发
componentWillUnmount    组件卸载前
componentWillMount  render挂载之前调用

22.类组件中更新时的钩子函数是什么？可以去改变状态吗？

shouldComponentUpdate 不可以会发生页面闪烁，会导致无限循环调用更新

23.路由的使用

Router中包含了对路径改变的监听，并且会将相应的路径传递给子组件
BrowserRouter是history模式，HashRouter是hash模式
使用两者作为最顶层组件包裹其他组件

24.useRef的使用

我们可以将useRef返回值看作一个组件内部全局共享变量，它会在渲染内部共享一个相同的值。相对state/props他们是独立于不同次render中的内部作用域值。
useRef返回值的改变并不会引起组件重新render
在React.functionComponent中想要获取对应jsx的真实Dom元素时候也可以通过useRef进行获取到对应的Dom元素。

25.在render函数使用this.setstate，会怎么样？

会触发死循环导致内存崩溃

26.自定义Hook？

自定义 hook 必须要以 use 开头命名遵循驼峰命名，在其内部调用其他 hook
自定义 Hooks 的核心是共享组件之间的逻辑
使用自定义 Hooks 能够减少重复的逻辑

27.路由跳转方式？怎么传递参数？

navlink link history history
params 以冒号拼接，参数会暴露在url地址栏
search 以？号拼接，参数也会暴露在url地址栏

29.setstate是同步还是异步？

在组件生命周期或React合成事件中，setState是异步
在setTimeout或者原生dom事件中，setState是同步

30.路由懒加载的使用

React.lazy 接受一个函数作为参数，这个函数需要调用import()。
React.lazy 方法返回的是一个lazy组件的对象，类型是react.lazy，并且 lazy 组件具有 _status 属性。并且与 promise 类似，有 Pending，Resolved，Rejected 三个状态，分别代表组件的加载中，已加载、加载失败三个状态。

31.vue与react的区别？

一、核心思想不同
Vue的核心思想是尽可能的降低前端开发的门槛，是一个灵活易用的渐进式双向绑定的MVVM框架。
React的核心思想是声明式渲染和组件化、单向数据流，React既不属于MVC也不属于MVVM架构。
二、组件写法上不同
Vue的组件写法是通过template的单文件组件格式。
React的组件写法是JSX+inline style，也就是吧HTML和CSS全部写进JavaScript中。
三、Diff算法不同
四、响应式原理不同
React的响应式原理
React主要是通过setState()方法来更新状态，状态更新之后，组件也会重新渲染。
Vue的响应式原理
vue会遍历data数据对象，使用Object.definedProperty()将每个属性都转换为getter和setter，每个Vue组件实例都有一个对应的watcher实例，在组件初次渲染的时候会记录组件用到了那些数据，当数据发生改变的时候，会触发setter方法，并通知所有依赖这个数据的watcher实例调用update方法去触发组件的compile渲染方法，进行渲染数据。
五、封装程度不同
封装程度，vue封装程度更高，内置多个指令和数据双向绑定，react封装度比较低，适合扩展。

32.hashrouter与browserouter的区别？

底层原理不一样：
BrowserRouter调用的是H5 history API,低版本兼容性问题。
HashRouter 使用的是URL哈希值
地址栏表现形式不一样：
BrowserRouter的路径：localhost:3000/demo/a
HashRouter的路径：localhost:3000/#/demo/a
刷新后对路由state参数的影响
BrowserRouter没有任何影响，因为state保存在history对象中。
HashRouter刷新后会导致路由state参数的丢失！！！

33.react中key的作用

具有唯一标识，可以根据diff算法更准确更快速的找到对应的元素

什么是组件化

组件化就是把可以复用的、独立的、基础的、功能专一的代码封装到一个方法或者代码片段里，在未来需要的地方引入使用。用极少的代码实现之前相同的功能，避免了相同功能代码的复写，提高了开发的效率。

什么是模块化

模块化是为了单独实现某一功能模块进行封装的方法，一个模块里可能拥有n个基础组件搭配产生。模块化属于横向分块，每个模块像一条横向把n条竖直的线串联起来形成一个整体。

thissetstate不可以在哪里使用

componentWillMount中执行,组件只挂载一次，在 componentWillMount里 setState 会但是仅会更新 state 一次，
componentWillUnmount中执行 setState 不会更新 state，是不生效而且无意义的
禁止在 shouldComponentUpdate 和 componentWillUpdate中调用setState，这会造成循环调用，
render函数也不行，会触发死循环导致内存崩溃

hooks使用有哪些限制

不能在循环语句, 条件语句, 或者嵌套函数中调用 Hooks 函数,
仅在函数组件和自定义 hooks 函数中调用 hooks 函数 不能在其他普通函数中调用

react和vue的相似和差异有哪些

区别：
vue 通过Templating模板渲染
react 通过JSX渲染
vue是双向绑定
react是单项绑定
单向绑定：插值形式， v-bind也是单向绑定。
双向绑定：表单v-model，用户对view层的更改会直接同步到model层。（实际上v-model只是v-bind:value 和 v-on:input的语法糖）
Vue是MVVM模式的一种方式实现。但没有完全遵循 MVVM 模型。
React是前端组件化框架，是严格的view层，MVC模式。由后端组件化发展而来。
相似：
都是单向数据流
组件化
虚拟dom

更新阶段的钩子函数中，改变状态应该怎么处理

进行if判断，否则会陷入递归更新而报错
可以将上一次的props（prevProps）和本次接收到的props做一个比较后再调用
